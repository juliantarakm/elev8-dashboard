Inbox Layout Approaches

Three approaches for structuring the Elev8 Inbox module

A

Adapt Mail Module Pattern

Reuse the existing mail/Layout.vue resizable 3-panel pattern. Swap in Inbox-specific content: conversation filters on left, thread list center-left, message thread center-right, reservation summary as a collapsible right panel or tab inside the thread view.

Pros

  • Fastest to build — leverages familiar layout
  • Consistent with existing codebase patterns
  • ResizablePanelGroup already wired up

Cons

  • Couple Reservation Summary into thread panel — may feel cramped
  • Mail code wasn't designed for 4 panels
B

4-Panel Resizable Layout

Four independent resizable panels: Filter sidebar | Conversation list | Message thread | Reservation summary. Each panel can collapse/expand. Built fresh but following the same ResizablePanelGroup pattern from mail/.

Pros

  • Matches PRD layout exactly
  • Each panel fully independent
  • Reservation context always visible

Cons

  • More complex resizable logic (4 panels)
  • Horizontal space is tight on <1440px screens
  • More UI to build from scratch
C

3-Panel + Slide-over Detail

Three main panels: Filter sidebar | Conversation list | Message thread. Reservation summary opens as a slide-over Sheet from the right when needed. Default collapsed, click to open.

Pros

  • Clean default view — not cluttered
  • Reservation detail on-demand
  • Works well on 1280px screens
  • Follows shadcn Sheet pattern

Cons

  • Reservation context not always visible
  • Extra click to see reservation details
