A) Side Panel (Drawer)

Panel slide dari kanan, halaman tetap visible di kiri. User bisa lihat tab content sambil edit resource.

Pros

  • Context tetap terjaga
  • Bisa lihat field yang akan di-fill
  • Familiar pattern (seperti settings drawer)

Cons

  • Makan space horizontal

B) Modal / Dialog

Dialog full-focus, halaman di-blur. User fokus ke resource, lalu klik "Auto-fill" untuk apply ke form.

Pros

  • Fokus penuh ke resource
  • Lebih banyak space untuk konten

Cons

  • Tidak bisa lihat field yang di-fill
  • Harus tutup modal untuk cek hasilnya

C) Split View (50/50)

Halaman dibagi dua — kiri tetap tab content, kanan panel Resources. Auto-fill langsung visible side-by-side.

Pros

  • Paling powerful — lihat source & target sekaligus
  • Auto-fill terasa "live"

Cons

  • Sempit di layar kecil
  • Lebih complex

💡 Rekomendasi: Option A (Side Panel) — paling balance antara context dan space. User bisa lihat tab yang akan di-fill sambil resource panel terbuka di kanan.
